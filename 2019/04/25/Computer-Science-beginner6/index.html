NAT(Network address translation) vs NAPT(Network address port translation)

NAT

  • 라우터가 가진 공인 IP 주소를 하나씩 할당해줌 (소수의 IP주소를 가지고 있으므로 그 수가 모자람)

NAPT

  • 라우터가 가진 하나의 공인 IP 주소를 각각 다른 포트로 테이블을 만들어 호스트에게 할당. (같은 IP라도 포트의 번호가 다름)

Server

  • 특정 컴퓨터의 특정 프로세스 : socket이 많음
    • port: socket에 부여된 숫자
      • well-known port
    • socket
      • listening socket - 접속 요청할 때에 사용
      • data socket : 접속 후에 사용

TCP / UDP

  • TCP :
    • 신되로가 높음
    • 패킷 유실시 -> 재전송
    • 부담이 큼 (패킷을 보낸후 클라이언트가 받았는지 확인을 받아야 함)
  • UDP :
    • 신뢰도가 낮음
    • 패킷 유실시 -> 재전송 하지 않음
    • 부담이 적고 빠름 (동영상, 게임, 유튜브 등에서 사용)
    • 데이터 일부가 손실될 가능성이 있음

Application

  • HTTP(Hyper Text Transfer Protocol)
  • HTTPS- HTTP보다 보안이 더 강함
  • FTP
  • SMTP
  • DNS(Domain Name System) - Server IP를 사람이 읽기 쉽게 바꿔주는 프로토콜. 역으로 client가 도메인 이름으로 찾으면 이를 IP 주소로 바꿔 서버로부터 정보를 받아옴.

컨텐트 협상

  • 컨텐트 협상: client가 요청을 보내면 다양한 버전을 가지고 있는 server가 어떤 버전을 보낼것인지 결정하는 일. client가 요청한 자료가 없을 경우 대체제로 어떤 것을 보낼것인지에 대한 협상
    • 서버 주도 협상(server-driven) : 서버가 마음대로 정해서 대체제를 클라이언트에게 전달함
    • 에이전트 주도 협상(agent-driven) :
      클라이언트가 요구한 자료가 서버에 없을 경우 클라이언트에게 다시 리퀘스트를 요구함. 이를 통해서 클라이언트가 요구하는 대체제를 보냄. (두 번의 요청과 응답이 필요 -> 효율성이 올라감)
